# 📊 Bookstore Analytics with SQL

## 🔹 Overview

This project analyzes bookstore data using SQL to extract business insights like sales performance, customer behavior, inventory management, and marketing ROI.

In this project, we will guide you step-by-step through building a complete bookstore analytics solution using SQL. By the end of this project, you will:

* Identify best-selling books and revenue drivers
* Track and alert on low inventory stock
* Segment customers by behavior (RFM)
* Measure ROI on customer acquisition
* Present insights to stakeholders

## 🧰 Tools Used

* SQL (SQLite/PostgreSQL/MySQL)
* Subqueries, Joins, Aggregations
* Common Table Expressions (CTEs)

## 📂 Database Structure

The bookstore database includes four key tables that represents books, customers, orders and marketing efforts. Each table connects through primary and foreign keys to form a robust relational database.

| Table Name       | Description                          |
|----------------|--------------------------------------|
| Books          | Contains book details and stock      |
| Customers      | Customer info and signup date        |
| Orders         | Purchase transactions                |
| MarketingSpend | Customer acquisition cost            |


## 📈 Key Insights

* Identified best-selling books and revenue drivers
* Detected low inventory stock
* Performed RFM customer segmentation
* Evaluated marketing ROI
* Found returning and churned customers
